# 本番モデルの訓練・運用方針

## 概要

バックテストで確定したモデル構成（改善B 閾値0.55）を本番用に訓練し、pkl保存・運用する方針を定める。

## 確定モデル構成

| 項目 | 設定 | ADR |
|------|------|-----|
| アルゴリズム | LightGBM（2値分類） | 001 #3 |
| ホライゾン | N=8（8時間先） | 002 #2 |
| ラベル閾値 | 3 pips（±0.03円未満はNaN） | 002 #1 |
| 確率閾値 | 0.55 | 003 #4 |
| レジームフィルタ | ADX > 25 | 資料3 |
| 損切り | ATR(14) × 1.5 | 003 #1 |
| 利確 | ATR(14) × 2.0 | 003 #1 |
| レバレッジ | 5倍以下 | 003 #2 |
| DD停止 | 10% | 003 #3 |

## 設計判断

### 判断1: 訓練データ — 全期間を使用（拡大窓の最終状態）

**問題**: 本番モデルをどのデータで訓練するか

**選択肢**:
1. 全期間（2016年〜現在）で訓練
2. 直近5年のみ
3. 直近3年のみ

**決定**: 全期間（2016年〜現在）

**理由**:
- ADR 001 #6: 拡大窓方式。WF検証の最終ウィンドウと同じ条件
- 資料5（MUFG）: 「1990年〜予測対象まで」で成功。長い学習期間が有効
- 62,528本の全データを使うことで、複数レジーム（円安/円高、低ボラ/高ボラ）をカバー

### 判断2: 特徴量選定 — RF重要度で15個を固定

**問題**: 本番モデルの特徴量をどう決めるか

**決定**: 全データでRF変数重要度を算出し、上位15個を本番用に固定。選定結果をメタデータとしてpklに同梱。

### 判断3: ハイパーパラメータ — Optuna 50trialで最適化

**問題**: 本番モデルのHPをどう決めるか

**決定**: 全データの80%で訓練、20%で検証し、Optuna 50trialでシャープレシオ最大化。最適パラメータをpklに同梱。

### 判断4: 再訓練頻度 — 月次

**問題**: モデルをどの頻度で再訓練するか

**選択肢**:
1. 毎日
2. 毎週
3. 毎月
4. 手動

**決定**: 毎月（月初にデータ更新→再訓練→pkl上書き）

**理由**:
- WF検証が月次スライドで実施されたため、月次再訓練と整合
- 毎日/毎週は計算コストが高く、モデルの安定性が下がる
- 資料5: 「継続的に最新データを取り込み再学習」

### 判断5: 保存形式 — pickle + メタデータJSON

**決定**:
- `models/production_model.pkl` — LightGBMモデル本体
- `models/production_meta.json` — 特徴量リスト、HP、訓練日時、性能指標

## 本番モデルの訓練手順

```
1. データ更新: scripts/collect_data.py --update
2. 本番訓練: python scripts/train_production.py
   → RF重要度で特徴量15個選定
   → Optuna 50trialでHP最適化
   → 全データで最終訓練
   → models/production_model.pkl 保存
   → models/production_meta.json 保存
3. 検証: 過学習チェック（精度差<15%）
```
