# 過学習対策

## 概要

マーケットデータの予測において過学習は最大のリスクである（資料2,5が一致）。本ADRでは正則化方針、過学習検出、データ品質に関する判断を記録する。

## 前提: マーケットデータの特殊性

資料5（MUFG樋口）が指摘する画像データとマーケットデータの根本的な違い:

| | 画像データ | マーケットデータ |
|---|---------|-------------|
| 再現性 | 高い（同じ手書き文字は似た特徴を持つ） | **圧倒的に低い** |
| データ量 | 大量（MNIST: 70,000枚） | 限定的（1H足で年間約6,000本） |
| ノイズ識別 | ゴミデータが入っても識別しやすい | ノイズと信号の区別が困難 |
| tSNE結果 | クラスが明確に分離 | クラスが混在 |

→ 画像分野で有効な手法（大量データ投入、深いモデル、弱い正則化）はマーケットでは失敗する

## 設計判断

### 判断1: LightGBMの正則化 — Kaggleデフォルトより強め

**問題**: LightGBMのハイパーパラメータ探索範囲をどう設定するか

**選択肢**:
1. Kaggle一般的デフォルト（max_depth=-1, min_child_samples=20, reg_lambda=0）
2. やや強め（中間設定）
3. 強め（浅い木、大きい最小サンプル、強い正則化）

**決定**: 強め。以下のOptuna探索範囲を設定

```
max_depth:         [3, 6]       ← 浅め（Kaggleデフォルト: -1=無制限）
min_child_samples: [50, 200]    ← 多め（Kaggleデフォルト: 20）
num_leaves:        [15, 63]     ← 少なめ（Kaggleデフォルト: 31）
reg_lambda:        [1.0, 10.0]  ← 強め（Kaggleデフォルト: 0）
reg_alpha:         [0.1, 5.0]   ← 強め（Kaggleデフォルト: 0）
learning_rate:     [0.01, 0.1]  ← 小さめ（Kaggleデフォルト: 0.1）
n_estimators:      [100, 1000]  ← early_stoppingと組み合わせ
```

**理由**:
- 資料5（MUFG）: 「マーケットデータは画像データと比較して再現性が圧倒的に低い」。tSNEで次元圧縮しても画像のようにクラスが明確に分離しない。→ モデルが「ノイズのパターン」を学習するリスクが高い
- 資料5: 「チューニング（パラメータ調節して適合度を上げる方法）をあまりやりすぎると、過学習になりやすくなる」
- 資料2（Optimax）: 「過学習（オーバーフィッティング）が最大のリスク。過去データに過度に最適化されたAIは前例のない市場の急変に対応できない」

**トレードオフ**:
- **利点**: 過学習を抑制し、アウトオブサンプルでの汎化性能を優先
- **欠点**: 学習セット内の精度が低くなる。ただしマーケット予測では学習精度よりアウトオブサンプル精度が重要

### 判断2: 過学習検出 — 訓練/検証の精度差10%で警告

**問題**: 過学習が発生していることをどう検出し、どう対処するか

**選択肢**:
1. 精度差 > 5% で警告（厳しい）
2. 精度差 > 10% で警告（中間）
3. 精度差 > 15% で警告（緩い）
4. 自動検出 + 自動対処

**決定**: 訓練精度 - 検証精度 > 10% で過学習警告を出す

**理由**:
- 資料5: チューニングのやりすぎが過学習の原因。定量的な検出基準が必要
- 資料3（SIG-FIN）: LSTM(AR)の正答率が58.33%。マーケット予測では60%近辺が現実的な上限であり、訓練精度が70%を超えていれば過学習の可能性が高い
- 10%の基準: 検証精度55%のとき訓練精度65%以上なら警告。十分に保守的

**対処フロー**:
```
過学習検出時:
  1. 正則化パラメータを強化（reg_lambda, reg_alphaを増加）
  2. max_depthを1段階浅くする
  3. 特徴量をさらに削減（変数重要度下位を除外）
  4. 上記で解消しなければ学習データ量を確認
```

**トレードオフ**:
- **利点**: 過学習を定量的に検出し、体系的に対処できる
- **欠点**: 閾値10%は経験的な値であり、最適値はデータに依存

### 判断3: 人工データ事前学習 — 不採用

**問題**: 資料3の「AR(1)人工データ事前学習」アプローチを採用するか

**選択肢**:
1. 採用（特徴量の分布から人工データを生成し事前学習）
2. 不採用（閾値ラベルでノイズ除去の思想を代替）
3. 将来検討（パイプラインに拡張ポイントを残す）

**決定**: 不採用

**理由**:
- 資料3が成功した条件は**非常に特殊**: 共和分ペアのスプレッドがAR(1)過程に従うという**明確な構造的仮定**があり、パラメータ（α, σ）を推定できた。この仮定があったからこそ「本質的なダイナミクスのみを学習」できた
- USD/JPY単体の方向予測には、価格変動の生成過程を記述する既知の構造がない。AR(1)のような明確なモデルを仮定できないため、人工データ生成の根拠がない
- 資料3の**思想**（ノイズ除去→本質のみ学習）は、ADR 002 判断1の「閾値ラベル」（±X pips未満をNaN化）として既に反映済み

**トレードオフ**:
- **利点**: 実装の複雑さを回避。根拠のない人工データ生成による誤ったバイアスの導入を防ぐ
- **欠点**: 資料3が示した「人工データ学習 > 実データ直接学習」の精度向上効果を得られない

## 過学習対策の全体像（優先順位付き）

3つ以上の資料が一致する対策を最優先とする。

| 優先度 | 対策 | 根拠（資料数） | ADR |
|--------|------|-------------|-----|
| **最優先** | ウォークフォワード検証（拡大窓） | 資料2,3,5 が一致 | ADR 001 判断6 |
| **最優先** | 予測確率の閾値フィルタ | 資料4,5 が一致 | ADR 003 判断4 |
| **高** | 変数重要度による特徴量選択（25-30→10-15個） | 資料4,5 が一致 | ADR 002 判断3 |
| **高** | 閾値ラベル（ノイジーなサンプルを除外） | 資料3 の知見 | ADR 002 判断1 |
| **高** | LightGBMの強め正則化 | 資料5 の知見 | 本ADR 判断1 |
| **中** | 訓練/検証の精度差モニタリング | 資料5 の知見 | 本ADR 判断2 |
| **低** | tSNEによるデータ品質検証 | 資料5 の知見 | 将来検討 |
| **不採用** | 人工データ事前学習 | 資料3（条件が特殊） | 本ADR 判断3 |
