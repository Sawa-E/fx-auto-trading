# リスク管理・取引ルール

## 概要

予測モデルの出力を実際の取引判断に変換する際のリスク管理ルールを定める。資料4（FFAJ）が「勝率とRR比で収益の7割が説明される」と示しており、モデル精度と同等にリスク管理が重要である。

## 設計判断

### 判断1: 損切り/利確ライン — ATRベース（RR比 > 1 を保証）

**問題**: エントリー後の損切り・利確ラインをどう設定するか

**選択肢**:
1. ATRベース（ATR × 乗数で動的設定）
2. 固定pips（例: 損切り30pips / 利確60pips）
3. トレーリングストップ
4. 利確なし（時間ベースで決済）

**決定**: ATRベース。損切り = ATR(14) × 1.5、利確 = ATR(14) × 2.0（RR比 = 1.33）

**理由**:
- 資料1（Zenn）: 「毎回7,000円の損失」の原因として**損切りロジックの不備**が指摘。損切りの設計が収益に直結する
- 資料4（FFAJ）: 高収益群（第4四分位）は勝率74.9% + RR比5.79。低収益群（第1四分位）は勝率53.6% + **RR比0.68**。「勝率が高くてもRR比<1なら負ける」。→ **RR比 > 1 を構造的に保証する設計が必要**
- 資料4: 勝率とRR比には「ゆるやかな負の相関」がある。損小利大を目指すと損切り貧乏で勝率悪化、利確を急ぐとRR比悪化。ATR×乗数で両方のバランスを取る
- ATRベースならボラティリティに適応: 高ボラ時は幅広、低ボラ時は幅狭に自動調整

**トレードオフ**:
- **利点**: ボラティリティ適応、RR比を構造的に保証（利確乗数 > 損切り乗数）
- **欠点**: ATR期間（14）と乗数（1.5, 2.0）の最適値はデータ依存。バックテストで検証が必要

**乗数のチューニング**: Optunaの探索範囲に含め、シャープレシオ最大化の一環として最適化する
- 損切り乗数: 1.0〜2.5
- 利確乗数: 1.5〜4.0
- 制約: 利確乗数 > 損切り乗数（RR比 > 1 を保証）

### 判断2: レバレッジ制限 — 5倍以下

**問題**: ポジションサイズ（レバレッジ）の上限をどう設定するか

**選択肢**:
1. 1倍（レバレッジなし）
2. 5倍以下
3. 10倍以下
4. 25倍（国内FX上限）

**決定**: 5倍以下

**理由**:
- 資料4（FFAJ）のクラスター分析結果:
  - **高収益クラスター1（31.6%）**: レバレッジ低い、勝率高い → 安定的に利益
  - **低収益クラスター3（37.3%）**: レバレッジ高い、勝率低い → 大きく損失
  - 四分位分析でも**安定収益群（第3四分位）のレバレッジが5.40倍で最も低い**
- 資料4: 第4四分位（高収益群）はレバレッジ12.08倍だが、これはハイリスク・ハイリターン。中央値15.3%と平均値54.7%の乖離が大きく、一部の成功者に引き上げられている
- 資料2（Optimax）: 「リスク管理が成功の鍵」「損切りライン設定、資金管理、最大ドローダウン監視」

**トレードオフ**:
- **利点**: ドローダウンを抑制し、資金を保全。資料4の安定収益群の特徴に合致
- **欠点**: 収益の絶対額が限定される。ただし予測モデルの精度が確認されるまでは安全優先

### 判断3: モデル停止ルール — 最大ドローダウン10%で停止

**問題**: モデルが機能不全に陥った場合の安全装置をどう設計するか

**選択肢**:
1. 最大ドローダウン10%で停止
2. 最大ドローダウン20%で停止
3. 連続N回負けで停止
4. 停止ルールなし

**決定**: 累積ドローダウンが10%を超えた時点で取引を停止し、モデル再学習または手動介入を行う

**理由**:
- 資料2（Optimax）: 「コロナショックのような突発事象で機能不全に陥り、想定外の損失を招く」「AIパフォーマンスの定期的な監視・調整」「最大ドローダウン監視」を推奨
- 資料5（MUFG樋口）: 「インプットデータではカバーしきれない事象が起こった場合に当該モデルの信頼度が当然下がるため、AIがどのようなデータで予測しているかをファンドマネージャーが把握し、そこから逸脱したと判断されれば、AIによる運用を停止したり、学習データを更新したりする必要がある」
- 資料4: 低収益群の月間収益率中央値は-40%。DD 10%で早期に停止すれば、このレベルの損失を回避できる

**トレードオフ**:
- **利点**: 壊滅的な損失を防止。モデルの前提が崩れた場合の安全装置
- **欠点**: 正常な変動でも10%に達する可能性がある（偽陽性）。閾値は運用実績を見て調整

**停止後のアクション**:
1. 直近のウォークフォワード検証結果を確認
2. 市場環境の変化（レジームシフト）がないか確認
3. 必要に応じて学習データを更新し再訓練
4. 再訓練後のシャープレシオが基準を満たせば取引再開

### 判断4: 予測確率フィルタ — 閾値0.60（初期値）

**問題**: モデルの予測確率がどの水準以上なら取引を実行するか

**選択肢**:
1. 0.50（全シグナルで取引）
2. 0.55
3. 0.60
4. 0.65

**決定**: 初期値0.60。バックテストで0.50-0.70の範囲を探索し最適化

**理由**:
- 資料4（FFAJ）: 勝率が最重要変数（%IncMSE: 36.58）。確率フィルタで低信頼度のシグナルを除外することで、実効勝率を引き上げる
- 資料5（MUFG）: 「引っ掛かりの良さ」= ポイントとなる状況変化の局面でしっかり投資判断ができるか。全局面で取引するのではなく、確信度の高い局面に絞る
- 資料2（Optimax）: 「統計的優位性を持つ取引を繰り返す」= エッジのある取引のみ実行
- ADR 002 判断1の閾値ラベル（±X pips未満のノイズ除外）と合わせて二重のフィルタリング

**トレードオフ**:
- **利点**: 勝率向上、取引回数減少によるスプレッドコスト削減
- **欠点**: 取引機会が減少。閾値が高すぎるとほとんど取引しなくなる

## 取引ルールまとめ

```
エントリー条件:
  1. 閾値ラベルで訓練されたLightGBMが予測
  2. P(方向) > 0.60 の場合のみシグナル発行
  3. 累積DD < 10% であること

ポジションサイズ:
  レバレッジ 5倍以下

損切り/利確:
  損切り = ATR(14) × 1.5
  利確   = ATR(14) × 2.0
  → RR比 = 1.33（構造的に > 1）

停止条件:
  累積ドローダウン > 10% → 全ポジション決済、取引停止
```
