# 特徴量エンジニアリング詳細

## 概要

予測モデルに投入する25-30個の特徴量の具体的なリスト、計算方法、設計方針を記録する。ADR 002 判断3「広く→刈り込み」に基づき、まず全量を生成し、変数重要度で10-15個に絞る。

## 設計方針

### 資料3の核心的知見: 「定常的・有界な指標を重視」

資料3（SIG-FIN）が示した「予測しやすいものを予測する」の原則を特徴量設計に適用:
- **有界な指標**（0-1, 0-100等）は値の範囲が決まっており、平均回帰的
- **非定常な指標**（SMAの絶対値、MACDの絶対値等）は価格水準に依存し、時期によって全く異なる値をとる
- → 有界・正規化された指標を**全体の半数以上**占めるように設計

### 資料5の知見: 「意味のないデータを入れるな」

- 特徴量同士の冗長性に注意（RSIとStochRSIは相関が高い等）
- ただし変数重要度で刈り込むため、初期段階では広めに含める（ADR 002 判断3）

## 設計判断

### 判断1: 均衡乖離度・定常性特徴量を重視 — 全体の約1/4を配分

**問題**: 資料3の「有界・定常的な指標」をどの程度重視するか

**選択肢**:
1. 重視（6-8個、全体の約1/4）
2. 普通（3-4個、テクニカル指標と同等）

**決定**: 重視（8個）

**理由**:
- 資料3: 「共和分スプレッド（定常過程）の予測は価格予測より高精度」。定常的な特徴量はモデルが学習しやすい
- テクニカル指標の多くは有界（RSI: 0-100, BB位置: 0-1等）であり、価格そのものより予測しやすい
- 「指標が極端な値から均衡に戻る方向」は、資料3の平均回帰の考え方と合致

### 判断2: ラグ特徴量の深さ — 1-6期ラグ

**問題**: リターンのラグ特徴量をどの深さまで含めるか

**選択肢**:
1. 1-6期ラグ（6個）
2. 1-12期ラグ（12個、v1と同じ）
3. ラグなし

**決定**: 1-6期ラグ（6個）

**理由**:
- デイトレードで1H足の場合、6時間以上前の情報は古すぎる
- v1の12期ラグから半減し、特徴量数を抑制（ADR 004の過学習対策と整合）
- テクニカル指標自体が過去情報を含んでいるため（例: SMAは過去N期間の平均）、ラグの役割は補完的

## 特徴量の分類サマリー

| カテゴリ | 個数 | 有界/平均回帰の割合 |
|---------|------|-------------------|
| A. テクニカル指標 | 6 | 3/6 (50%) |
| B. 均衡乖離度 | 8 | **8/8 (100%)** |
| C. 時間帯 | 5 | 5/5 (100%) |
| D. モメンタム | 3 | 1/3 (33%) |
| E. ラグ | 6 | 0/6 (0%) |
| **合計** | **28** | **17/28 (61%)** |

全28個のうち61%が有界または平均回帰的。資料3の「定常的な系列を重視」を反映。

## v1からの変更点

| 変更 | 理由 |
|------|------|
| williams_r 削除 | StochRSIと冗長（両方オシレータ系） |
| sma_short/long/granville 削除 | 絶対値は非定常。代わりに close_sma_ratio（比率）を採用 |
| macd/macd_signal 削除 | macd_hist（差分）のみ残す。絶対値は非定常 |
| ラグ 12→6 に削減 | デイトレードで6時間超は古すぎる |
| regime_* 4個を削減 | adx, plus_di, minus_di で情報が含まれる |
| 均衡乖離度カテゴリ新設 | 資料3の知見を反映 |

## 刈り込みプロセス（ADR 002 判断3）

```
Step 1: 28個の特徴量を全て生成
Step 2: LightGBMの変数重要度（gain）を算出
Step 3: 上位15個を選定し再訓練
Step 4: 精度が維持or向上すれば15個版を採用
Step 5: さらに上位10個で同様に試行
```
