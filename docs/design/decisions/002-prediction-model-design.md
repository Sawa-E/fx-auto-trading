# 予測モデル設計

## 概要

USD/JPY デイトレード予測モデルの設計判断を記録する。5つのリサーチ資料の知見に基づき、予測形式・ホライゾン・特徴量戦略・最適化目標を決定する。

## 設計判断

### 判断1: 予測形式 — 2値分類 + 閾値ラベル + 確率フィルタ

**問題**: 何をどのような形式で予測するか
**選択肢**:
1. 2値分類（上/下）+ 確率閾値フィルタ
2. 3値分類（上昇/下落/横ばい）
3. 2値分類 + 閾値ラベル（±X pips未満をNaN化）+ 確率フィルタ
4. 回帰（変化率を直接予測）

**決定**: 選択肢3。訓練時に±X pips未満の動きをNaN化して学習から除外し、予測時に確率閾値で見送りも判断する

**理由**:
- 資料3（SIG-FIN）: AR(1)人工データで**ノイズを除去**した結果、LSTM(AR)の正答率が56.59%→58.33%に向上（+1.74%）。「予測しやすいものを予測する」が核心。閾値ラベルは同じ思想で、ノイジーなサンプルを学習から排除する
- 資料4（FFAJ）: 勝率が収益予測の最重要変数（%IncMSE: 36.58）。確率フィルタで高信頼度のシグナルのみ取引することで実効勝率を引き上げる
- 資料5（MUFG）: 3値分類（上昇/下落/横ばい）で成功しているが、「横ばい」の定義にドメイン知識が必要で恣意性が入る。閾値ラベルの方がシンプルで再現性が高い

**トレードオフ**: 閾値Xの設定が恣意的になるリスクがある。ATRベースで動的に設定するか、複数の閾値を実験して決定する。学習データが減るが、ノイズ除去による精度向上が上回ると期待

**実装詳細**:
```
訓練時:
  diff = close[t+N] - close[t]
  if diff > +X pips → label = 1（上昇）
  if diff < -X pips → label = 0（下落）
  else → NaN（学習から除外）

予測時:
  P(上昇) > 0.60 → 買いシグナル
  P(下落) > 0.60 → 売りシグナル
  else → 見送り
```

### 判断2: 予測ホライゾン — 複数ホライゾン実験（1/4/8足先）

**問題**: 何足先を予測対象にするか
**選択肢**: 1. 1足先（1時間） 2. 4足先（4時間） 3. 8足先（8時間） 4. 複数ホライゾンを実験

**決定**: 1/4/8足先の3パターンを実験し、シャープレシオが最も高いものを採用

**理由**:
- 資料3（SIG-FIN）: 20営業日先のスプレッド予測で成功。「長いホライゾンほどノイズが平滑化されて予測しやすい」
- 資料4（FFAJ）: 「保有期間の重要性は非常に低い」（%IncMSE: 7.14で最下位）。スキャル/デイトレ/スイングの区別は収益に直結しない
- 上記2つの知見から、最適なホライゾンは事前にはわからず、実験的に決定するのが最善

**トレードオフ**: 3パターンの実験は工数が3倍だが、ホライゾンは精度に直結するため投資対効果が高い

### 判断3: 特徴量戦略 — 広く生成 → 変数重要度で刈り込み

**問題**: 特徴量を何個使うか、どう選別するか
**選択肢**:
1. 最初から10個に厳選（資料5方式）
2. 25-30個生成 → 変数重要度で10-15個に刈り込み
3. 25-30個生成 → tSNEで予測力検証 → 刈り込み

**決定**: 選択肢2。25-30個を生成し、LightGBMの変数重要度で上位10-15個に機械的に刈り込む

**理由**:
- 資料5（MUFG）: 「AIに意味のないデータを入れる時代は終焉する」。10種類の加工データで成功。ただしこれは20年以上のマーケット経験を持つファンドマネージャーが厳選した結果
- 資料4（FFAJ）: ランダムフォレストの変数重要度（%IncMSE, IncNodePurity）で勝率>RR比>レバレッジ>保有期間の順位付けに成功。機械的な選別が有効であることを実証
- 我々にはファンドマネージャー級のドメイン知識がないため、まず広く入れて変数重要度で機械的に刈り込むのが安全

**特徴量の方針**（資料3の知見）:
- **有界・平均回帰的な指標を重視**: RSI(0-100), BB位置(0-1), チャネル位置(0-1), StochRSI(0-1)
- 資料3: 「定常的な系列は予測しやすい」。テクニカル指標の多くは有界で平均回帰的であり、価格そのものより予測しやすい

**トレードオフ**: 広く入れると過学習リスクが上がる。変数重要度による刈り込みとLightGBMの正則化で対処

### 判断4: 最適化目標 — シャープレシオ

**問題**: Optunaのハイパーパラメータ最適化で何を目的関数にするか
**選択肢**: 1. 正答率（Accuracy） 2. F1スコア 3. プロフィットファクター 4. シャープレシオ

**決定**: シャープレシオ（Sharpe Ratio = mean(returns) / std(returns)）

**理由**:
- 資料4（FFAJ）: 勝率（%IncMSE: 36.58）とRR比（33.12）が収益の2大決定要因。ただし「勝率が高くてもRR比<1なら負ける」（第1四分位群: 勝率53%、RR比0.68）。勝率だけの最適化は危険
- 資料2（Optimax）: リスク管理が成功の鍵。最大ドローダウン監視を推奨
- シャープレシオは mean(returns)/std(returns) であり:
  - 勝率が高い → meanが上がる
  - RR比が高い → meanが上がる
  - ドローダウンが大きい → stdが上がり、シャープレシオが下がる
  - → 資料4の「勝率×RR比」と資料2の「リスク管理」を自然に統合する唯一の指標

**トレードオフ**: 取引シミュレーション込みの計算が必要で、F1スコアより計算コストが高い。ただしOptunaの枝刈りで対処可能
