## 概要

三菱UFJ信託銀行のシニアファンドマネージャーが、実際に開発した為替AI予測モデルの技術的な構築方法と注意点を解説したレポート。ユーロドルの方向予測を題材に、複数の機械学習アルゴリズムを比較し、**XGBoost（Gradient Boosting系）が最も良好な結果**を示した実務的知見を報告。

---

## 核心的主張

### 1. 「AIに意味のないデータを入れる時代は終焉する」

- 数年前は「関係がありそうなデータを何でも入れるべき」という主張が多かった
- 画像・音声では正しい（再現性の高いデータが大量にあるためゴミを識別しやすい）
- **マーケットデータではこのアプローチは失敗する**
  - マーケットデータは画像データと比較して**再現性が圧倒的に低い**
  - tSNEで次元圧縮すると、画像データはクラスが明確に分離されるが、マーケットデータはクラスが混在する

### 2. AIモデル開発に必要な3つの知識

| 知識 | 内容 | 保有者 |
|------|------|--------|
| ①プログラミング知識 | コーディング能力 | プログラマー |
| ②機械学習メソッドの数学的知識 | 統計・ML構造の理解 | データサイエンティスト |
| ③予測対象データの知識 | マーケットの理解 | ファンドマネージャー |

- ①②を持つデータサイエンティストは一定数存在
- ①②③を同時に持つ人材が**圧倒的に少ない**
- **マーケット知識を有するファンドマネージャーにデータサイエンスを教育するのが一番の近道**

### 3. 学習データの未来予測力の検証方法

- **tSNE**（t分布型確率的近傍埋め込み法）による次元圧縮で学習データの質を視覚化
- 10種類の加工データ（日米金利差、為替変動等）を2次元・3次元に圧縮
- ドル円ヘッジコストの上昇/下落/横ばいのラベルがある程度分離されることを確認
- **マーケットデータはPCAではなくtSNEを使用すべき**（非線形データが多いため）

## 為替AI予測モデルの実装

### 使用した機械学習アルゴリズム一覧

| カテゴリ | アルゴリズム |
|---------|----------|
| カーネル法 | サポートベクターマシン（SVM） |
| ツリーモデル・アンサンブル学習 | ランダムフォレスト、Gradient Boosting、**LightGBM**、**XGBoost** |
| ニューラルネットワーク | ディープラーニング、CNN、RNN、LSTM |

### モデル構築プロセス

```
Step 1: 計量分析手法で学習データの選別を行う
        ↓
Step 2: 3種のベーシックモデルで簡易予測
        - サポートベクターマシン（SVM）
        - ランダムフォレスト
        - ディープラーニング
        ↓
Step 3: 最も「引っ掛かりが良い」モデルを選定
        ↓
Step 4: 上位モデル（XGBoost等）で精度向上
```

**「引っ掛かりが良い」の定義**: 精度が高いという意味ではなく、**ポイントとなる状況変化の局面でAIがしっかり投資判断ができているか**。

### 実験設定

| 項目 | 値 |
|------|------|
| 対象 | ドルユーロ（USDEUR） |
| 予測対象 | 翌週の方向性（上昇/下落/横ばい） |
| 学習データ | 1990年1月〜予測対象の1つ手前まで |
| 検証方法 | 2009年10月から逐次推定（**アウトオブサンプル予測**） |
| 入力データ | 日米金利差や為替の変動など加工データ10種類 |
| ラベル | 3値分類（上昇○、下落▲、横ばい×） |

### モデル比較結果

| モデル | 結果 | 評価 |
|--------|------|------|
| **ランダムフォレスト** | 正答割合に改善余地あるが、**変化の局面で反応できている** | 引っ掛かり良好 |
| ディープラーニング | 2016年以降、相場が大きく動いているにも関わらず**殆ど投資行動を取っていない** | 不良 |
| SVM | **下落中心の予測**になってしまい、変化の局面にうまく反応できていない | 不良 |
| **XGBoost** | ランダムフォレストから**かなり精度が上がった** | 最良 |

### XGBoostが最良だった理由

- ランダムフォレストの上位モデル（Gradient Boostingとランダムフォレストの組み合わせ）
- チューニング項目は増えるが、全般的に精度が高い
- **Kaggleでも LightGBM と並んで最も人気のあるアルゴリズム**
- 「実は人気上位はディープラーニングではない。ディープラーニングは必ずしも万能なモデルではない」

### ディープラーニングの限界

- 標準的なディープラーニング（全結合NN）は**系列相関の情報を学習する能力がない**
- 時系列情報を扱うにはCNN/RNN/LSTMを使うか、学習データに時系列情報（差分データ、移動平均等）を入れる必要がある
- **大量のデータを必要とする**（マーケットデータでは不足しがち）

## 今後の展望（著者の見解）

### AIは運用者を代替するか？

**答え: NO**

- AIモデルの予測精度は高いが、「自動運転」までできるわけではない
- インプットデータでカバーしきれない事象が起こった場合、モデルの信頼度が下がる
- **AIとファンドマネージャーはお互い補完し合う関係**
  - ファンドマネージャー: AIがどのようなデータで予測しているか把握し、逸脱を判断
  - AI: 人間では検知できないパターンを発見

### 完全自動化への距離

- 現時点: AIにインプットデータでカバーしきれない事象かどうかを判断させ、自らデータを見つけて学習する「完全自動化」にはまだ距離がある
- 将来: ファンドマネージャーが自分の相場観を反映させたAIモデルとともに運用する時代は遠くない

## 本プロジェクトへの示唆

| 知見 | 適用 |
|------|------|
| **XGBoost/LightGBMがマーケット予測で最良** | LightGBMの選択を強く支持。ディープラーニングに走る必要なし |
| 意味のないデータを入れるな | 計量分析で特徴量を事前選別。変数重要度で不要な特徴量を除外 |
| マーケットデータは再現性が低い | 過学習リスクが画像分野より格段に高い。正則化を強めに |
| 「引っ掛かりの良さ」が重要 | 精度だけでなく、**変化の局面でシグナルを出せるか**を評価 |
| tSNEでデータの質を可視化 | 特徴量の予測力を事前検証する手法として採用を検討 |
| 3値分類（上昇/下落/横ばい） | 2値（上/下）だけでなく「横ばい＝取引しない」の3値分類も検討価値あり |
| 計量分析で学習データを選別 | ML前にファンダメンタルズ分析で入力データを絞る |
| AIは万能ではなく補完ツール | モデルの限界を理解し、異常時に停止する仕組みを設計 |

### 特に重要な設計インプット

#### 3値分類の検討

現在の設計は2値分類（上/下）だが、本資料は3値分類（上昇/下落/**横ばい**）を示唆している:
- 横ばい = 予測が難しい + スプレッドコストで負ける
- 横ばいと予測されたときは**取引しない**
- これは「予測確率の閾値フィルタ」と同様の効果を持つ

#### モデル構築プロセスの採用

1. まずSVM、ランダムフォレスト、ディープラーニングの3モデルで簡易比較
2. 「引っ掛かりの良い」モデルを選定
3. その上位モデル（LightGBM/XGBoost）で精度向上

→ このプロセスは本プロジェクトでも採用する価値がある

#### tSNEによる特徴量の予測力検証

特徴量生成後、tSNEで次元圧縮し、ラベル（上昇/下落）がある程度分離されるか視覚的に確認する工程を入れる。分離が見られなければ、その特徴量セットには予測力がない。

## 付録: 各アルゴリズムの特性比較

| アルゴリズム | 長所 | 短所 | マーケット適性 |
|------------|------|------|------------|
| ランダムフォレスト | 欠損値に強い、チューニング少、使い勝手良い | 精度にやや限界 | 良好（ベースライン） |
| XGBoost/LightGBM | 高精度、Kaggle人気No.1 | チューニング項目多い | **最良** |
| SVM | 少ないデータでも良い結果、過学習に比較的強い | 高次元化で過学習リスク | 中程度 |
| ディープラーニング（全結合） | 柔軟で強力 | 大量データ必要、系列相関学習不可 | 低い |
| LSTM/RNN | 時系列の長期依存を学習可能 | 大量データ必要、訓練困難 | 場合による |
