## 概要

FX投資家の個人取引データ（2015年1-2月、各月1,000人サンプル）を用いて、クラスター分析とランダムフォレストによりパフォーマンスの決定要因を分析した研究。**勝率が収益率予測において最も重要な変数**であることを実証的に示した。

---

## データ

| 項目 | 値 |
|------|------|
| 提供元 | 金融先物取引業協会の会員FX会社 |
| サンプル期間 | 2015年1月〜2月（2ヶ月間） |
| サンプル数 | 月ごとに1,000人をランダムサンプリング |
| 市場環境 | 2013-2014年の急激な円安が一服し、トレンドが掴めない時期 |
| 通貨ペア | USD/JPYを含む13通貨ペア |

### サンプリング条件

1. 月に1回以上取引をしている投資家
2. 前月末時点でポジションを保有していない投資家
3. 月初に預託証拠金を差し入れている投資家

---

## 分析に使用した変数

### 月間収益率

含み損益を考慮した月次口座評価額収益率。計算式:

```
日次口座評価額グロス収益率t = (有効証拠金t + 出金額t) / (有効証拠金t-1 + 入金額t)

入出金額t = 預託証拠金t - 預託証拠金t-1 - (確定損益t + スワップ損益t)

月間収益率 = (Π 日次グロスリターンt - 1) × 100(%)
```

### 保有期間

| 分類 | 定義 |
|------|------|
| スキャル投資家 | 保有期間1時間以内 |
| デイトレ投資家 | 保有期間1日以内 |
| スイング投資家 | 保有期間1日超 |

分類方法: 各投資家の全取引を通貨ごとに分類し、ある分類が仕切約定回数の5割を超える場合、その分類を当該投資家の保有期間とする。

### レバレッジ

```
月次レバレッジ = 日次レバレッジ(必要証拠金t × 25 / 有効証拠金t) の月間平均
```

NYクローズ時のオーバーナイトレバレッジ。レバレッジがゼロの日は平均から除外。

### 勝率

仕切注文ベースの取引数のうち、実現益を出した取引の比率。

### リスクリワード比

```
リスクリワード比 = 1取引当たりの平均実現益 / 1取引当たりの平均実現損
```

勝率100%と0%の投資家は算出不能のため除外。

#### 読み取れるパターン

- **高収益（第4四分位）**: 勝率70%超、デイトレ比率が高い
- **低収益（第1四分位）**: レバレッジ16倍、勝率50%台、リスクリワード比が1未満
- **安定収益（第3四分位）**: レバレッジが最も低い（5-6倍）、勝率60-70%
- **勝率が高くてもリスクリワード比<1なら負ける**: 第1四分位群は勝率53%あるがRR比0.68

### 結果2: 勝率とリスクリワード比の関係

- 勝率とリスクリワード比には**ゆるやかな負の相関**がある
- 損小利大を目指すと損切り貧乏になって勝率が悪化
- 勝率を上げようと利確を急ぐとリスクリワード比が悪化
- **低収益群**: 勝率が高くてもリスクリワード比が1未満の投資家が多い
- **高収益群**: 勝率もリスクリワード比も両方高い（稀だが存在する）

### 結果4: クラスター分析（階層型、ウォード法）

5変数（収益率、保有期間、レバレッジ、勝率、リスクリワード比）を標準化してウォード法で階層的クラスタリングを実施。デンドログラムから最適クラスター数は**3**と判明。

| クラスター | 人数（割合） | 構成 | 特徴 |
|-----------|------------|------|------|
| **1** | 219人（31.6%） | スキャル中心 | **収益率高い、レバレッジ低い、勝率高い** |
| **2** | 215人（31.1%） | デイトレ・スイング中心 | 収益率中程度、レバレッジ中間、勝率中程度 |
| **3** | 258人（37.3%） | スキャル中心 | **収益率低い、レバレッジ高い、勝率低い** |

**重要な発見**: スキャル投資家は特性の異なる2つのグループが混在しており、保有期間だけで分類することは不適切。同じスキャルでも:
- クラスター1型: 低レバレッジ × 高勝率 → 高収益
- クラスター3型: 高レバレッジ × 低勝率 → 低収益

### 結果5: ランダムフォレストによる変数重要度（最重要結果）

**被説明変数**: 月間収益率
**説明変数**: レバレッジ、勝率、リスクリワード比、保有期間

| 変数 | %IncMSE | IncNodePurity | 順位 |
|------|---------|---------------|------|
| **勝率** | **36.58** | **66.01** | **1位** |
| **リスクリワード比** | **33.12** | **58.22** | **2位** |
| レバレッジ | 14.87 | 47.21 | 3位 |
| 保有期間 | 7.14 | 9.05 | 4位 |

**指標の意味**:
- **%IncMSE**: ある説明変数をランダムに置換したときMSE（平均二乗誤差）がどれだけ増加するか。高いほど重要
- **IncNodePurity**: ある説明変数がランダムに置換される前後のRSS（残差平方誤差）の差から算出されるジニ係数。高いほど重要

---

## 論文の結論

1. **勝率が最も重要**: 収益率予測において勝率が圧倒的に重要（%IncMSE: 36.58）
2. **リスクリワード比が2番目**: 勝率×リスクリワード比の組み合わせが収益を決定（%IncMSE: 33.12）
3. **レバレッジは3番目**: リスクの代理変数として一定の影響（%IncMSE: 14.87）
4. **保有期間はほぼ無関係**: スキャル/デイトレ/スイングの区別は収益に直結しない（%IncMSE: 7.14）
5. **スキャル投資家は二極化**: 高勝率・低レバレッジ群と低勝率・高レバレッジ群が混在

## 本プロジェクトへの示唆

| 知見 | 適用 |
|------|------|
| **勝率が最重要** | モデルの予測精度（正答率）を最優先で最適化する |
| **リスクリワード比が2番目** | 予測確率の閾値を設けて高信頼度のシグナルのみ取引 → RR比向上 |
| **レバレッジは低く** | 高レバレッジは低収益群の特徴。低レバレッジ × 高勝率が正解 |
| **保有期間は関係ない** | 1時間足で問題なし。時間足の選択で悩む必要はない |
| **勝率が高くてもRR比<1なら負ける** | 損切り・利確のロジックが勝率と同等に重要 |
| **高収益群: 勝率70%超 + RR比>1** | これが目指すべきターゲット |

### モデル設計への具体的適用

1. **予測確率でフィルタ**: LightGBMの出力確率が閾値（例: 0.60-0.65）を超えるときのみシグナル発行 → 実効勝率の引き上げ
2. **ATRベースの損切り/利確**: RR比 > 1 を保証する損切り・利確ライン設計
3. **レバレッジ制限**: 低レバレッジ（5-6倍以下）を前提とした資金管理
4. **評価指標**: 精度だけでなく勝率 × RR比 × シャープレシオの組み合わせで評価
