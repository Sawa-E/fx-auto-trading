# ステータスドキュメントガイド

## 1. 目的

設計書から実装ステータスを分離し、進捗を一元管理する。

**対象読者**: 開発者、AIエージェント

## 2. ファイル構成

| ファイル | 内容 |
|---------|------|
| `implementation.md` | モジュール別の実装ステータス |
| `roadmap.md` | Phase別のロードマップ |

## 3. ステータス値

| 値 | 意味 |
|----|------|
| `done` | 実装完了・テスト済み |
| `partial` | 一部実装済み |
| `not-started` | 未着手 |

## 4. 更新タイミング

- モジュールの実装が完了したとき
- Phase が進行したとき
- 新しいモジュールが追加されたとき
