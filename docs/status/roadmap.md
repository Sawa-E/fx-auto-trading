# ロードマップ

## Phase 1: 予測モデル構築（精度最優先）

データ取得 → 特徴量生成 → モデル訓練 → ウォークフォワード検証

- GMO FX API から USD/JPY 1H OHLCデータ収集
- テクニカル指標ベースの特徴量エンジニアリング
- LightGBM による方向予測モデル
- ウォークフォワード検証で汎化性能を確認
- 簡易バックテストで実用性を評価

## Phase 2: バックテスト・リスク管理

予測モデルをベースにした売買戦略の構築と検証

- 売買シグナル生成ロジック
- リスク管理（損切り・利確・ポジションサイジング）
- バックテストエンジン
- 性能レポート

## Phase 3: 自動売買

GMO FX Private API を使った実取引

- ペーパートレード
- 実取引実行
- モニタリング・通知

### 必須課題: 大暴落時のリスク管理

現在のPhase 2（仮想取引）では損切りは1H足の高値/安値で「事後計算」しているだけで、即時執行はできない。Phase 3では以下を実装する必要がある:

1. **OCO注文（逆指値）**: エントリーと同時にSL/TPの逆指値を取引所に出す → 取引所側で自動決済（スリッページは残るがギャップ以外は対応可能）
2. **最大損失制限**: 1取引あたりの最大損失を口座残高の2-3%に制限
3. **同時ポジション上限**: 複数ポジションの同時保有を制限
4. **日次損失リミット**: 日次損失が一定額を超えたら全ポジション決済+当日停止
5. **ギャップリスク対応**: 週末持ち越し禁止（金曜NY引け前に全決済）

## Phase 4: 継続改善サイクル

モデルの劣化検知と自動改善

### 第1層: 監視（自動・毎日）— 最優先
- 直近N取引のPF・勝率をトラッキング
- 劣化検知アラート（PF < 1.3等）をDiscordへ通知

### 第2層: 自動再訓練（月1回）
- 最新データで同じパイプラインを再実行
- 新旧モデルのバックテスト成績を自動比較
- 上回った場合のみ差し替え

### 第3層: 構造改善（四半期・手動）
- 特徴量・閾値・ハイパーパラメータの見直し
- 新しい論文・手法の取り込み
- 売りモデルの改善検討
