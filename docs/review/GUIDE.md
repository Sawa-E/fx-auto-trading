# コードレビューガイド

## 1. 目的

本ガイドは fx-auto-trading のコードレビューにおける手順・基準を定める。

**対象読者**: 開発者、AIエージェント

## 2. ディレクトリ構成

```
review/
├── GUIDE.md          # 本ガイド
├── FORMAT.md         # 共通出力形式
├── TEMPLATE.md       # モジュールレビューテンプレート
└── modules/          # モジュール別チェックリスト
```

## 3. 役割分担

| ファイル | 内容 |
|---------|------|
| GUIDE.md | レビューの全体方針・手順 |
| FORMAT.md | レビュー結果の出力形式 |
| TEMPLATE.md | 個別モジュールのチェックリストテンプレート |
| modules/*.md | モジュール固有の評価項目 |

## 4. レビュー手順

1. 対象モジュールの設計書を確認（`design/modules/`）
2. ソースコードを通読
3. `modules/{module}.md` のチェックリストに沿って評価
4. FORMAT.md に従って結果を記録

## 5. 記述ルール

- 問題は具体的なコード参照（`file.py:line` または `file.py:ClassName.method()`）と共に記述
- 修正提案は具体的なコード例を含める
- 「良い点」も記録する（確認済みセクション）

## 6. モジュール追加手順

1. `TEMPLATE.md` をコピーして `modules/{module-name}.md` を作成
2. モジュール固有の評価項目を追加
3. 本ガイドの対象モジュール一覧を更新

## 7. 対象モジュール一覧

初期状態では未定義。モジュール実装後に追加する。
