# Phase 2: フォワードテスト（仮想取引）

## 概要

確定モデル（改善B, N=8, 閾値0.55）を使い、プロジェクト内の仮想取引エンジンで完全自動運用する。GitHub Actionsで毎時実行し、Discordでエントリー/決済/損益を通知。1ヶ月間の結果でPhase 3（リアル取引）への移行を判断する。

---

## これまでの成果

### リサーチ（5つの論文・レポート）

| # | 資料 | 主な知見 |
|---|------|---------|
| 1 | Zenn ML短期投資 | 損切りロジックが鍵。単純な実装では損失が出る |
| 2 | Optimax AIトレーディング | 過学習が最大リスク。フォワードテスト必須 |
| 3 | SIG-FIN 共和分ペアトレード | 定常的な系列は予測しやすい。ノイズ除去で精度向上 |
| 4 | FFAJ FXパフォーマンス | 勝率が最重要。RR比>1必須。保有期間は無関係 |
| 5 | MUFG マーケットAI予測 | XGBoost/LightGBMが為替で最良。意味のないデータを入れるな |

### 設計判断（7つのADR）

| ADR | テーマ | 決定 |
|-----|-------|------|
| 001 | 技術スタック | Python/uv/LightGBM/SQLite/Optuna |
| 002 | 予測モデル | 2値+閾値ラベル(3pips)、N=8、特徴量28→15個、シャープレシオ最適化 |
| 003 | リスク管理 | ATR×1.5/2.0損切利確、レバ5倍、DD10%停止、確率閾値0.55 |
| 004 | 過学習対策 | 強め正則化(max_depth3-6)、精度差15%警告 |
| 005 | データパイプライン | 初回一括+差分更新、ohlcのみ、2016年〜 |
| 006 | 特徴量 | 34個(テクニカル+均衡乖離+時間帯+モメンタム+ラグ+delta+MTF) |
| 007 | 本番モデル | 全データ訓練、月次再訓練、pkl+json保存 |

### 確定モデル構成

| 項目 | 設定 |
|------|------|
| モデル | LightGBM（`models/production_model.pkl`） |
| 特徴量 | RF重要度上位15個（`models/production_meta.json`） |
| ホライゾン | N=8（8時間先の方向予測） |
| ラベル閾値 | ±3pips未満はNaN化（ノイズ除去） |
| 確率閾値 | 0.55（P≥0.55で買い、P≤0.45で売り） |
| レジームフィルタ | ADX > 25（トレンド相場のみ） |
| 損切り | ATR(14) × 1.5 |
| 利確 | ATR(14) × 2.0（RR比1.33） |

## Phase 2 の設計

### 仮想取引エンジン

外部デモ口座を使わず、プロジェクト内で仮想取引を完結させる。

```
毎時（GitHub Actions）:
  1. GMO APIから最新価格を取得
  2. オープンポジションのSL/TPチェック → ヒットしたら決済
  3. 新しいシグナル判定 → 条件を満たせばエントリー
  4. Discord通知（エントリー/決済/日次サマリー）
  5. 状態ファイルをgit commit & push（永続化）
```

### 仮想口座の設定

| 項目 | 値 | 根拠 |
|------|-----|------|
| 初期資金 | 30,000円 | ユーザー指定 |
| レバレッジ | 5倍 | ADR 003 #2 |
| ポジションサイズ | 150,000円分 ≈ 約940通貨 | 30,000×5÷159 |
| DD停止ライン | -3,000円（残高27,000円） | ADR 003 #3（10%） |
| スプレッド | 0.3pips | ADR 003 |
| 同時ポジション | 1つまで | リスク制限 |

### Discord通知の内容

```
エントリー時:
🟢 エントリー: 買い USD/JPY @ 159.14
  SL: 158.69 | TP: 159.74 | サイズ: 940通貨
  確率: 0.58 | ADX: 32.5
  残高: 30,000円

決済時（利確）:
💰 利確ヒット! +564円
  買い 159.14 → 159.74
  残高: 30,564円 (+1.9%) | 通算: 5勝1敗

決済時（損切り）:
📉 損切り -430円
  買い 159.14 → 158.69
  残高: 29,570円 (-1.4%) | 通算: 5勝2敗

見送り時（通知なし）:
  → Discordは静か
```

### GitHub Actionsワークフロー

```yaml
毎時1分（平日UTC）:
  1. checkout（virtual_account.json含む）
  2. pip install -e .
  3. python scripts/predict.py
     → 価格取得 → SL/TPチェック → 新シグナル判定
     → Discord通知 → virtual_account.json更新
  4. git commit & push（状態永続化）
```

## 成功基準（1ヶ月後に判定）

| 指標 | バックテスト | 最低基準 | 判定後のアクション |
|------|-----------|---------|-----------------|
| 勝率 | 60.1% | 55%以上 | |
| PF | 2.01 | 1.5以上 | |
| 取引頻度 | 3.5回/日 | 1回/日以上 | |
| DD | 1.1% | 10%未満 | 10%超で自動停止 |

**全基準クリア → Phase 3（GMO Private APIでリアル取引）に移行**
**PF<1.5 → モデル再訓練・改善を検討**
