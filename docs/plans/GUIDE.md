# 実装計画ガイド

## 1. 目的

実装計画はギャップ分析・優先度付け・スコープ定義・作業計画の記録に使用する。

**対象読者**: 開発者、AIエージェント

## 2. ディレクトリ構成

```
plans/
├── GUIDE.md          # 本ガイド
├── TEMPLATE.md       # 実装計画テンプレート
├── {計画名}.md       # アクティブな実装計画
└── resolved/         # 完了済み計画
```

### ライフサイクル

1. `plans/` に新規計画を作成（アクティブ）
2. 完了後、`plans/resolved/` に移動

## 3. 実装計画の種別

| 種別 | 用途 | 例 |
|------|------|-----|
| `enhancement` | 新機能追加 | 予測モデル構築 |
| `bugfix` | バグ修正 | API接続タイムアウト対策 |
| `refactoring` | リファクタリング | モデル訓練コードの分離 |
| `missing_docs` | ドキュメント追加 | モジュール設計書の作成 |

## 4. ドキュメント構成

| セクション | 必須/推奨 | 説明 |
|-----------|----------|------|
| メタデータコメント | 必須 | 種別・優先度・ステータス |
| 概要 | 必須 | 何を・なぜ実装するか |
| スコープ | 必須 | 含むもの・含まないもの |
| タスクリスト | 必須 | チェックボックス形式 |
| 技術的な考慮事項 | 推奨 | 注意点・制約 |
| 関連ドキュメント | 必須 | 参照リンク |

## 5. 記法規約

### メタデータ

```markdown
<!--
種別: enhancement
優先度: high | medium | low
ステータス: draft | active | completed
作成日: YYYY-MM-DD
-->
```

### タスクID

タスクには一意のIDを付与する: `TASK-{計画番号}-{連番}`

### タスクリスト

```markdown
- [ ] TASK-001-01: データ収集クラスの実装
- [ ] TASK-001-02: APIクライアントのテスト
```

### スコープ境界

```markdown
## スコープ

**含む**:
- GMO FX APIからのOHLCデータ取得
- SQLiteへのデータ保存

**含まない**:
- リアルタイムWebSocket接続
- 他ブローカーAPI対応
```

## 6. AIエージェント向けルール

- 計画作成前に関連する設計書・ADRを確認する
- タスクは1つの実装単位（1PR相当）に分割する
- 依存関係がある場合はタスクの順序を明記する
- 完了したタスクはチェックを付けるが、計画自体は完了後に `resolved/` へ移動する

## 7. 品質チェックリスト

- [ ] スコープが明確に定義されている
- [ ] タスクが具体的で実行可能な粒度になっている
- [ ] 依存関係が明記されている
- [ ] 関連ドキュメントへのリンクがある

## 8. テンプレートの使用

`TEMPLATE.md` をコピーし、計画名で保存する。

命名規則: `{3桁番号}-{kebab-case}.md` （例: `001-prediction-model.md`）
