※サンプル数16回は統計的に少なく、この数値の信頼性は低い

## 考察

### 良い点
- 全指標が成功基準をクリア（取引頻度以外）
- 14勝2敗（87.5%）はバックテスト（60.1%）を大幅に上回る
- 2回の負けでも損失が限定的（-156円、-173円）。ATR損切りが機能
- 全てbuyシグナル → 2-3月の円安トレンドを正しく捕捉

### 注意点
- **サンプル数16回は少なすぎる**。統計的に有意とは言えない
- **全てbuyシグナル**。売りシグナルの性能は未検証
- **取引が集中**（3/2に7連続取引）。同一方向の連続エントリーはリスク
- 取引頻度0.8回/日は目標の1回/日にわずかに届かない
- **月利24%は異常に高い**。長期的にはバックテスト水準（月3-5%）に収束する見込み

### バックテストとの乖離について
- 勝率がバックテスト（60.1%）よりフォワード（87.5%）が高いのは、**直近1ヶ月が明確なトレンド相場**だったため
- レジームフィルタ（ADX>25）がトレンド相場を正しく捕捉し、高勝率に寄与
- レンジ相場が来たときに勝率が下がることは想定内

## 追加検証: 円高期間（2022年10-11月）

### テスト設計

| 項目 | 内容 |
|------|------|
| 訓練データ | 2016-01-03 ~ 2022-09-30（42,003本） |
| テストデータ | 2022-10-02 ~ 2022-11-30（1,034本） |
| 市場環境 | 日銀為替介入、151円→137円の急激な円高（-7.30円） |

### 結果

| 指標 | 値 |
|------|-----|
| 取引回数 | **0回** |
| 買いシグナル | 0回 |
| 売りシグナル | **0回** |

### 原因分析

全1,034本の予測確率が **0.513〜0.515** の極めて狭い範囲に集中。モデルは「わからない」と答え続けた。

```
予測確率の分布（2022/10-11, 1034本）:
  平均: 0.514
  最小: 0.513
  最大: 0.515
  → 全てが見送り範囲（0.45-0.55）に収まる
```

ADX>25の期間（759本、72.9%）でも同様。閾値を0.52, 0.51まで下げても売りシグナルは0回。

### 論文に基づく解釈

**資料5（MUFG）**:
> 「インプットデータではカバーしきれない事象が起こった場合に当該モデルの信頼度が当然下がる」

2022年10月の日銀介入は訓練データに存在しない異常イベント。モデルが「自信がない」と判断し取引を見送るのは、**安全装置として正しい挙動**。

**資料2（Optimax）**:
> 「前例のない市場の急変に対応できない」

まさにこのケース。

### 問題点

| 良い点 | 問題点 |
|--------|--------|
| 異常イベントで取引しない → 損失を回避 | 売りシグナルが一切出ない → 下落相場で利益を取れない |
| 資料5の「逸脱したら停止」を自動実現 | 訓練データのラベルバランス（上昇52.7%）による買いバイアスの可能性 |

## 発見された課題と改善案（論文ベース）

### 課題1: 売りシグナルが機能しない

**現象**: 円高期間でモデルの確率出力が0.51付近に固定され、売りシグナル（P≤0.45）が出ない

**根本原因**:
- 訓練データのラベルバランスが上昇52.7% / 下落47.3%と上昇に偏っている
- 拡大窓方式で2016年〜全データを使用 → 古いレジームのデータが多く、最新のパターンが希釈される
- 強い正則化（max_depth=3）により、モデルが保守的すぎる出力をする

**資料3（SIG-FIN）の知見**:
> 「パラメータαが時間により変化することに注意を要する。従来研究ではαを一定として扱い、予測精度の低下をもたらしていた」

→ 拡大窓は「全期間で一定のパターン」を前提にしている。市場レジームの変化に対応するには、**直近の学習データを重視**する必要がある。

**改善案A: スライディングウィンドウ（固定窓）に変更**
- 拡大窓（2016年〜全部）→ 直近2-3年のスライディングウィンドウ
- 古いレジームのデータを捨て、最新のパターンに適応
- ADR 001 #6の変更が必要

**改善案B: 買い/売り別モデル**
- 現在: 1つのモデルで上昇確率を出力
- 改善: 「買いモデル」と「売りモデル」を別々に訓練
- 売りモデルは下落トレンドのデータを重点的に学習
- 資料5: 「全く構造が違うアルゴリズムは同じ誤りを犯しづらい」の応用

**改善案C: 月次再訓練を週次に変更**
- ADR 007では月次再訓練と決定
- 週次にすれば、レジーム変化後1週間で新パターンを学習可能

### 課題2: 確率出力の分散が小さい

**現象**: 確率が0.51付近に集中し、0.55以上/0.45以下がまれ

**資料5（MUFG）の知見**:
> 「マーケットデータは画像データと比較して再現性が圧倒的に低い」

→ LightGBMが保守的な出力をするのはデータの性質上仕方ない面がある。ただし改善の余地はある。

**改善案D: LightGBMの確率キャリブレーション**
- Platt scalingやisotonic regressionで確率出力を補正
- 0.51付近に集中する出力を、より広い範囲に分散させる

## 結論

**本番デプロイに進めてよい。ただし「買い専用」として運用する。**

模擬フォワードテストで買いシグナルの有効性を確認（14勝2敗）。一方、円高期間では売りシグナルが出力されないことが判明。

### 本番デプロイ方針

1. **Phase 2は「買い専用」で運用開始** — 買いシグナルの実績は十分
2. **売りシグナルの改善は並行して検討** — 改善案A-Dを評価し、Phase 2.5で対応
3. **円高トレンドが来た場合** — モデルが見送りを続ける = 損失は出ない（安全）

### 優先度の高い改善案

| 優先度 | 改善案 | 内容 | 期待効果 |
|--------|-------|------|---------|
| 高 | A: スライディングウィンドウ | 直近2-3年のデータで訓練 | レジーム追従性↑、売りシグナル出力 |
| 高 | D: 確率キャリブレーション | 確率出力の分散を拡大 | シグナル発生率↑ |
| 中 | B: 買い/売り別モデル | 方向別に特化したモデル | 売り精度↑ |
| 低 | C: 週次再訓練 | 月次→週次に変更 | レジーム変化への即応 |
